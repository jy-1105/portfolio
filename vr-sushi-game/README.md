# VR_寿司弾丸

## 概要
VR空間で、来店した客の注文に応じて寿司を作り、投げて提供するゲームです。  
シャリとネタを両手でつかんで組み合わせ、完成した寿司を客に届けます。  
注文どおりに提供するとスコアを獲得でき、プレイを妨害する特殊な客も登場するため、状況判断が求められるゲーム性を取り入れました。

## 開発期間
2025年11月〜2026年1月

## 開発体制
4人チーム開発

## 使用技術
- Unity
- C#
- XR Interaction Toolkit
- NavMesh

## 担当部分
- VR環境における寿司の生成・把持・投擲処理の実装
- 注文UI・タイマーUIの実装
- 客のスポーン、空席判定、座席移動、着席処理の実装
- 客アニメーションの制御
- スコアシステムおよび注文成功・失敗判定の実装
- リザルト画面におけるランク表示や演出の実装

## 工夫した点
- プレイヤーが説明なしでも操作方法を理解できるよう、シャリとネタを両手でつかみ組み合わせる直感的な操作設計を行いました。  
- 注文内容や残り時間を瞬時に把握できるよう、頭上UIの配置や視認性を考慮した設計を行いました。  
- 客が生成されてから席に移動し、着席後に注文を開始する一連の流れを実装することで、ゲーム全体の進行を自然に感じられるようにしました。  
- プレイ後にも達成感や喜びを感じられるよう、スコア集計、ランク表示、リプレイを含むリザルト演出を工夫しました。

## 課題と解決
### 課題
客が生成された後、空いている席を判定し、自然に移動・着席・注文開始までつなげる必要がありました。

### 解決
座席ごとに使用状態を持たせ、スポナー側で空席を探索する仕組みを実装しました。  
さらに、NavMeshAgentを用いて席まで移動させ、到着後は位置・向きを座席に合わせた上で着席アニメーションを再生し、注文処理を開始する流れを構築しました。  
これにより、プレイヤーが違和感なく状況を理解できる自然な挙動を実現しました。

- [CustomerOrderWithTimer.cs](./scripts/CustomerOrderWithTimer.cs)  
  注文生成、タイマーUI、正誤判定、スコア反映を担当

- [ResultManager.cs](./scripts/ResultManager.cs)  
  スコア集計、ランク表示、リプレイ表示を含むリザルト演出を管理
